# Minecraft 服务器管理项目

这个仓库用于维护“你好新蒸程”长期服务器的管理面板、自动化脚本、部署配置和运维文档。服务器仅供少量成员使用，因此流程保持轻量，但所有重要修改都应可追溯、可验证、可恢复。

## 从这里开始

- 普通成员和服主先阅读 [成员使用说明](docs/user/codex-workflow.md)。
- 当前服务器概况见 [服务器概况](docs/user/server-overview.md)。
- 完整备份部署包见 [备份与百度网盘上传](ops/backup/README.md)。
- 第一次部署和交接见 [备份系统交接手册](ops/backup/HANDOFF.md)。
- 正式服冷备份、Minecraft 自动恢复、百度网盘分卷续传和唯一自动 timer 已完成首次生产验收并启用；
  当前状态及维护命令见 [备份 README](ops/backup/README.md)。
- 历史部署和故障记录见 [历史记录](docs/history/README.md)。
- Codex 会自动读取根目录的 `AGENTS.md`，成员无需把其中规则复制到每次对话。

## 仓库范围

仓库保存：

- 服务器信息面板源代码。
- 自动备份、监控、维护和人员管理脚本。
- systemd 服务和应用配置模板。
- 成员说明、操作手册和脱敏后的变更记录。

仓库不保存：

- SSH、面板、网盘或第三方服务密码。
- 世界存档、服务器备份和运行日志原件。
- 服务端或客户端整合包压缩文件。
- Spark 原始报告和临时下载文件。

这些本地资料统一保存在被 Git 忽略的 `.local/` 或现有制品目录中。

## 协作方式

小型、低风险修改可以直接提交到 `main`。涉及停服、重启、世界数据、认证模式、端口或玩家权限的修改，应先向服主说明影响并获得明确确认。每次远程写操作完成后，需要在历史记录中留下脱敏摘要。
