# 正式服务器远程操作规则

## 默认权限边界

无需额外确认即可执行只读检查，包括：服务状态、进程、端口、资源使用、玩家数量、配置读取、日志读取和文件校验。只读检查也应避免高负载全盘扫描。

以下操作需要用户在当前任务中明确授权：

- 停止、重启或强制终止 Minecraft 服务。
- 修改 `server.properties`、认证模式、白名单、端口、防火墙或端口映射。
- 修改 OP、封禁、玩家数据、世界数据或模组配置。
- 安装或删除软件、模组、世界文件和 systemd 服务。
- 执行区块预生成、批量实体清理、恢复备份等可能造成性能波动的任务。

用户要求部署或修改某项功能时，可以执行实现该功能所必需且影响范围清晰的写操作；若其中包含上述高风险动作，仍需提前说明影响。

## 写操作前

1. 检查目标服务是否运行、当前在线人数和系统资源。
2. 确认准确的目标路径，不根据模糊匹配批量覆盖或删除。
3. 修改重要配置前制作带时间戳的局部备份。
4. 优先使用可重复执行的脚本或仓库内配置，而不是临时手工命令。
5. 不在命令行、日志或文档中回显密码和令牌。

## 写操作后

1. 验证命令退出状态、服务状态、端口和相关日志。
2. 不使用 Minecraft 客户端代替用户进行进服测试；需要客户端验证时明确告知用户测试步骤。
3. 在 `docs/history/` 的对应日志中记录日期、目标、改动文件、验证结果和回滚方式，不记录秘密。
4. 将服务器上的最终脚本、配置和 systemd 单元同步回仓库。

## 运行中服务保护

- 不以“顺便更新”为理由重启正式服。
- 性能诊断优先使用现有指标、短时采样和 Spark 报告，避免无边界的高开销采集。
- 玩家在线时避免备份压缩、区块预生成、大范围查找和实体批处理；确有必要时先告知影响。
- 不承诺零影响。应准确说明预期负载、持续时间和可回滚性。
