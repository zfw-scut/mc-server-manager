# 与 Codex 协作

成员可以直接用自然语言描述目标，不需要预先了解 Linux、Minecraft 服务端或项目代码。为了减少误操作，需求最好包含目标、允许的影响和验证方式。

## 推荐写法

```text
检查服务器当前 TPS 和内存，只做只读分析，不要重启。
```

```text
给面板增加在线玩家列表。允许更新并重启面板服务，但不要操作 Minecraft 服务。
```

```text
部署自动备份脚本。先完成代码和配置，不要立即启用定时任务，等我确认。
```

## 需要特别说明的操作

涉及以下内容时，请明确说明是否允许执行：

- 停止或重启 Minecraft 服务。
- 修改正版验证、白名单、OP、封禁或玩家数据。
- 修改世界、模组、端口和防火墙。
- 执行区块预生成、批量实体清理或备份恢复。

没有明确授权时，Codex 默认只检查和分析，不执行这些高影响操作。

## 完成结果

Codex完成任务后应说明：修改了什么、是否影响在线玩家、如何验证、是否需要玩家客户端测试，以及相关记录保存在哪里。秘密信息不会写入 GitHub 文档。
